Guía interactiva — Wallet con jQuery + Bootstrap

Este sitio explica y demuestra el comportamiento requerido sin entregar la solución completa. Úsalo para entender eventos, validaciones, Local Storage, redirecciones y UI con Bootstrap.

Eventos LocalStorage Validaciones Bootstrap Alerts Scroll triggers

1) Cómo usar esta guía en clase

  • No contiene los 5 HTML finales ni el código exacto de la wallet.
  • Sí muestra qué eventos capturar, qué datos guardar, cómo debería verse la interacción.
  • Incluye demos activadas por scroll y por tiempo.
Objetivo didáctico: que el alumno comprenda el “qué” y el “por qué”, y programe el “cómo”.

2) login.html — Lo que debe ocurrir (sin implementación)

En login, el alumno debe usar selectores jQuery para leer email y password, capturar el submit con jQuery, validar, mostrar alertas Bootstrap y redirigir al menú.

Checklist de comportamiento submit
  • Leer valores con $('#email').val() y $('#password').val().
  • Evitar recarga con e.preventDefault().
  • Validar: campos vacíos, formato email, longitud mínima password.
  • Mostrar alert Bootstrap: éxito o error (sin alert() nativo).
  • Redirigir al menú si OK.
Ejemplo de “plantilla mental”

Esto es pseudocódigo, no es tu solución final:

// pseudoflujo
onSubmit(loginForm):
  leer email/password (jQuery)
  validar
  si error -> mostrar alerta Bootstrap
  si ok -> guardar sesión (opcional) -> redirigir

3) menu.html — Botones y redirecciones

“Depositar”, “Enviar dinero”, “Últimos movimientos”: al click debe aparecer una leyenda temporal tipo “Redirigiendo a …” y luego navegar a la pantalla correspondiente.

4) deposit.html — Saldo, depósito, alertas y redirección con delay

Al cargar: leer saldo desde Local Storage y mostrarlo. Al depositar: actualizar saldo, mostrar leyenda del monto depositado, alerta de éxito y redirigir al menú tras 2 segundos.
